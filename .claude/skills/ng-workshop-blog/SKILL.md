## Instructions

### Step 1: Understand the Topic

Ask the user:
1. What is the blog topic/title?
2. Which content pillar does it belong to?
3. Who is the primary persona (CTO, COO, CISO, Process Owner)?
4. Any specific keywords to target?
5. Reference materials or sources to include?

### Step 2: Research (If Needed)

Before writing, research:
- Current trends and data on the topic
- Competitor content on similar topics
- Israeli market specifics
- Relevant case studies or examples

### Step 3: Create Outline

Create a structured outline following this format:

```markdown
# Blog Outline: [Title]

## Hook / Opening
- [Compelling opening - statistic, question, or story]
- [Why reader should care]

## Main Sections (3-5)
- Section 1: [Problem/Context]
- Section 2: [Core Content/Solution]
- Section 3: [How to Implement/Examples]
- Section 4: [NG Workshop Angle - Camunda connection]

## Conclusion
- Summary
- CTA to contact

## SEO
- Primary keyword:
- Secondary keywords:
- Meta description:
```

### Step 5: Apply Writing Standards

#### Voice & Tone
- Professional but accessible (not academic)
- First person plural ("we", "us") when mentioning NG Workshop
- Direct address to reader ("you", "your organization")
- Hebrew that sounds natural, not translated

#### Structure
- Short paragraphs (2-4 sentences max)
- Use bullet points and tables for lists
- Include code examples for technical content (BPMN, XML)
- Visual breaks every 3-4 paragraphs

#### SEO Optimization
- Primary keyword in title, H1, first paragraph
- Secondary keywords in H2 headers
- Meta description under 160 characters
- 5-7 relevant tags
- Keywords section at end

#### NG Workshop Integration
- Natural mention of Camunda capabilities
- Reference to "Process-First" approach
- Human-in-the-Loop connection when relevant
- CTA to contact page
- Don't be overly promotional
