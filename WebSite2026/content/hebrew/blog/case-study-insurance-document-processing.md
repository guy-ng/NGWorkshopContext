חברות ביטוח מטפלות במיליוני מסמכים מדי שנה: תביעות, פוליסות, אישורים רפואיים, ועוד. עיבוד ידני איטי, יקר, ומועד לטעויות. הנה איך פתרנו את זה עם IDP + Process Orchestration.

## הרקע

**הארגון:** חברת ביטוח גדולה
**התהליך:** עיבוד תביעות ביטוח בריאות
**היקף:** 50,000+ תביעות בחודש

### המצב לפני:
- עיבוד ידני של כל מסמך
- זמן עיבוד ממוצע: 7 ימים
- שיעור שגיאות: 8%
- צוות של 40 פקידים

## האתגר

### 1. מגוון מסמכים עצום
- קבלות מרופאים (אלפי פורמטים)
- אישורים רפואיים
- הפניות
- חשבוניות בתי חולים
- מסמכי פוליסה

### 2. עברית + ערבית + אנגלית
- מסמכים בשפות שונות
- כתב יד לפעמים
- סריקות באיכות משתנה

### 3. חוקים מורכבים
- כל סוג תביעה עם כללים שונים
- בדיקת כיסוי פוליסה
- חישוב השתתפות עצמית

## הפתרון: IDP Pipeline

### ארכיטקטורה:

```
┌─────────────────────────────────────────────────────────────┐
│                   קליטת מסמכים                              │
│     [אימייל] [אפליקציה] [פקס] [סניפים]                     │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│              שלב 1: Pre-Processing                          │
│  • שיפור איכות תמונה                                        │
│  • יישור וחיתוך                                             │
│  • זיהוי כיוון (עברית RTL)                                  │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│              שלב 2: סיווג מסמכים                            │
│  • AI מזהה סוג מסמך                                         │
│  • ניתוב לעיבוד המתאים                                      │
│  • זיהוי מסמכים חסרים                                       │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│              שלב 3: חילוץ נתונים                            │
│  • OCR מתקדם (עברית + ערבית)                                │
│  • NER - זיהוי ישויות                                       │
│  • טבלאות וסכומים                                           │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│             שלב 4: אימות ובדיקות                            │
│  • Cross-validation בין שדות                                │
│  • בדיקת תקינות מול פוליסה                                  │
│  • זיהוי חריגות                                             │
└──────────────────────────┬──────────────────────────────────┘
                           ↓
┌─────────────────────────────────────────────────────────────┐
│            Camunda Process Orchestration                    │
│  • ניתוב לפי סוג תביעה                                      │
│  • חישוב פיצוי                                              │
│  • Human-in-the-Loop לחריגות                                │
│  • אישור ותשלום                                             │
└─────────────────────────────────────────────────────────────┘
```

### שלב 1: Pre-Processing

**שיפור איכות:**
- ניקוי רעשים
- הגברת ניגודיות
- יישור אוטומטי

**דוגמה:**
```
קלט: סריקה מטושטשת, נייר מקומט
     ↓
עיבוד: שיפור, יישור, חיתוך
     ↓
פלט: תמונה חדה ומיושרת
```

### שלב 2: סיווג מסמכים

**AI מזהה סוג מסמך:**

| סוג | דיוק זיהוי |
|-----|-----------|
| קבלה מרופא | 98% |
| אישור רפואי | 96% |
| חשבונית בית חולים | 97% |
| מרשם | 94% |
| הפניה | 95% |

**טיפול במסמכים לא מזוהים:**
- העברה לבדיקה ידנית
- למידה מהמקרים החדשים
- שיפור מתמיד של המודל

### שלב 3: חילוץ נתונים (OCR + NER)

**שדות שמחלצים מקבלה:**
```json
{
  "provider_name": "ד״ר יוסי כהן",
  "provider_id": "12345678",
  "date": "2025-01-15",
  "treatment_type": "ביקור רופא מומחה",
  "diagnosis_code": "J06.9",
  "amount": 450.00,
  "patient_name": "ישראל ישראלי",
  "patient_id": "123456789"
}
```

**אתגרים ופתרונות:**

| אתגר | פתרון |
|------|--------|
| כתב יד | מודל מיוחד לכתב יד עברי |
| סכומים מטושטשים | Cross-validation עם סה"כ |
| תאריכים בפורמטים שונים | Normalization אוטומטי |
| שפות מעורבות | מודל multi-lingual |

### שלב 4: אימות ובדיקות

**בדיקות אוטומטיות:**
1. **עקביות פנימית** - סה"כ = סכום פריטים?
2. **תקינות** - האם הספק מורשה?
3. **כיסוי** - האם הטיפול כלול בפוליסה?
4. **חריגות** - האם הסכום חריג?

**מנגנון Confidence Score:**
```
ציון ביטחון > 95%  → אישור אוטומטי
ציון ביטחון 80-95% → בדיקה מהירה
ציון ביטחון < 80%  → בדיקה מלאה
```

## תהליך העבודה ב-Camunda

### תביעה פשוטה (Straight-Through Processing):

```
[מסמכים] → [IDP] → [בדיקת כיסוי] → [חישוב] → [אישור] → [תשלום]
     ↓                                              ↓
  2 דקות                                        10 דקות
```

### תביעה מורכבת:

```
[מסמכים] → [IDP] → [בדיקה ידנית] → [אישור מנהל] → [תשלום]
     ↓           ↓                    ↓
  2 דק'      30 דק'                2 שעות
```

### מסך פקיד לבדיקת חריגות:

```
┌──────────────────────────────────────────────────────────┐
│              תביעה #78901 - לבדיקה                       │
├──────────────────────────────────────────────────────────┤
│ מבוטח: יעל לוי | פוליסה: 123456 | סוג: ביטוח בריאות      │
├──────────────────────────────────────────────────────────┤
│                                                          │
│ [תמונת מסמך]        │ נתונים שחולצו:                     │
│ ╔═══════════════╗   │ ספק: ד״ר משה כהן ✓                │
│ ║  קבלה         ║   │ תאריך: 15/01/2025 ✓               │
│ ║  ד״ר משה כהן  ║   │ סכום: ₪850 ⚠️ (גבוה מהממוצע)      │
│ ║  ...          ║   │ סוג טיפול: ייעוץ מומחה ✓          │
│ ╚═══════════════╝   │                                    │
│                     │ ציון ביטחון: 87%                   │
├──────────────────────────────────────────────────────────┤
│ ⚠️ התראות:                                              │
│ • סכום גבוה ב-40% מהממוצע לסוג טיפול זה                 │
│ • תביעה שנייה החודש מאותו ספק                           │
├──────────────────────────────────────────────────────────┤
│ [אשר ₪850] [אשר חלקי ₪600] [דחה] [בקש מסמכים נוספים]   │
└──────────────────────────────────────────────────────────┘
```

## התוצאות

### מדדים כמותיים:

| מדד | לפני | אחרי | שיפור |
|-----|------|------|-------|
| זמן עיבוד (פשוט) | 7 ימים | 15 דקות | **-99.8%** |
| זמן עיבוד (מורכב) | 14 ימים | 2 ימים | **-85%** |
| שיעור STP | 0% | 65% | **חדש** |
| שגיאות | 8% | 1.2% | **-85%** |
| עלות לתביעה | ₪35 | ₪12 | **-66%** |
| צוות נדרש | 40 | 15 | **-62%** |

### מדדים איכותיים:

**שביעות רצון לקוחות:**
- "קיבלתי את הכסף תוך שעה!"
- פחות שיחות למוקד
- NPS עלה ב-25 נקודות

**שביעות רצון עובדים:**
- פחות עבודה שגרתית
- יותר מקרים מאתגרים
- הרגשת ערך

## גורמי הצלחה

### 1. איכות נתוני אימון
- אלפי דוגמאות מתויגות
- מגוון פורמטים וספקים
- עדכון שוטף

### 2. Feedback Loop
- כל תיקון ידני חוזר למודל
- שיפור מתמיד
- ירידה בהתערבות ידנית לאורך זמן

### 3. Human-in-the-Loop נכון
- לא מנסים להחליף הכל
- אנשים במקרים הנכונים
- כלים טובים לפקידים

### 4. מדידה ושיפור
- KPIs ברורים
- דשבורדים בזמן אמת
- זיהוי צווארי בקבוק

## לקחים

### מה עבד:
✅ התחלה עם סוג מסמך אחד והרחבה
✅ שיתוף הפקידים המנוסים בתיוג
✅ Confidence score גבוה לפני STP
✅ מנגנון ערעור פשוט ללקוח

### מה היינו עושים אחרת:
⚠️ יותר השקעה באיכות סריקות מההתחלה
⚠️ אינטגרציה מוקדמת עם ספקים גדולים
⚠️ הכשרה מעמיקה יותר לצוות

## סיכום

IDP + Process Orchestration שינו את תהליך התביעות:

- **מהירות** - מימים לדקות
- **דיוק** - פחות שגיאות אנוש
- **עלות** - חיסכון של מיליונים
- **חוויה** - לקוחות מרוצים

המפתח: לא "להחליף אנשים במכונות" אלא לתת לכל אחד לעשות את מה שהוא טוב בו.
