לפני שמשפרים תהליך - צריך להבין אותו. מיפוי תהליכים הוא הצעד הראשון בכל פרויקט אוטומציה או שיפור. הנה מדריך מעשי ב-5 צעדים.

## למה למפות תהליכים?

- **להבין** מה באמת קורה (לא מה אנחנו חושבים שקורה)
- **לזהות** צווארי בקבוק ובזבוזים
- **לתקשר** בשפה אחידה עם כל הגורמים
- **לתכנן** שיפורים בצורה מדויקת

## צעד 1: הגדרת גבולות התהליך

### מה לקבוע?
- **שם התהליך:** מה אנחנו ממפים?
- **טריגר:** מה מתחיל את התהליך?
- **תוצאה:** מה התוצר הסופי?
- **גבולות:** מה בפנים ומה בחוץ?

### תבנית:

```
┌─────────────────────────────────────────┐
│          הגדרת תהליך                    │
├─────────────────────────────────────────┤
│ שם: ______________________________      │
│                                         │
│ טריגר (מה מתחיל את התהליך):             │
│ _____________________________________   │
│                                         │
│ תוצאה סופית:                            │
│ _____________________________________   │
│                                         │
│ מה כלול:                                │
│ □ _____________________________________│
│ □ _____________________________________│
│                                         │
│ מה לא כלול:                             │
│ □ _____________________________________│
│ □ _____________________________________│
│                                         │
│ בעל התהליך: ____________________        │
└─────────────────────────────────────────┘
```

### דוגמה: תהליך אישור הוצאה

```
שם: אישור הוצאה
טריגר: עובד מגיש בקשה להחזר
תוצאה: כסף מועבר לעובד או בקשה נדחית

כלול:
✓ הגשת הבקשה
✓ אישור מנהל
✓ העברה לכספים
✓ ביצוע תשלום

לא כלול:
✗ רכישה של העובד (לפני ההגשה)
✗ דיווח לרשויות המס
```

## צעד 2: זיהוי השחקנים

### מי מעורב בתהליך?
- **אנשים:** עובדים, מנהלים, לקוחות
- **מערכות:** ERP, CRM, אימייל
- **יחידות ארגוניות:** מחלקות, צוותים

### תבנית:

| שחקן | תפקיד בתהליך | מערכות שמשתמש |
|------|--------------|---------------|
|      |              |               |
|      |              |               |

### דוגמה: תהליך אישור הוצאה

| שחקן | תפקיד | מערכות |
|------|-------|--------|
| עובד | מגיש בקשה | פורטל עובדים |
| מנהל ישיר | מאשר/דוחה | אימייל + פורטל |
| כספים | מבצע תשלום | מערכת ERP |
| מערכת | שולחת התראות | אימייל אוטומטי |

## צעד 3: מיפוי הצעדים

### איך לאסוף מידע?

**1. ראיונות עם מבצעים:**
- "תאר לי מה קורה כשמתחיל התהליך"
- "מה עושים אחר כך?"
- "מה קורה כשיש בעיה?"

**2. צפייה בעבודה:**
- שבו ליד עובד וצפו
- תעדו את מה שרואים, לא מה שמספרים

**3. מסמכים קיימים:**
- נהלי עבודה
- טפסים
- הודעות מערכת

### תבנית לתיעוד צעדים:

```
צעד #___

מבצע: _________________
פעולה: _________________
קלט: __________________
פלט: __________________
מערכת: ________________
זמן משוער: _____________
מה יכול להשתבש: ________
```

### דוגמה: תהליך אישור הוצאה

```
צעד #1
מבצע: עובד
פעולה: מגיש בקשה + מצרף קבלה
קלט: קבלה, פרטי ההוצאה
פלט: בקשה פתוחה במערכת
מערכת: פורטל עובדים
זמן: 5 דקות
בעיות: קבלה לא ברורה, פרטים חסרים

צעד #2
מבצע: מערכת
פעולה: בודקת תקציב + שולחת למנהל
קלט: בקשה
פלט: התראה למנהל
מערכת: ERP + אימייל
זמן: מיידי
בעיות: מנהל לא מוגדר

צעד #3
מבצע: מנהל
פעולה: בודק ומאשר/דוחה
קלט: בקשה + פרטים
פלט: אישור/דחייה
מערכת: אימייל או פורטל
זמן: 1-3 ימים (!)
בעיות: מנהל בחופשה, שוכח לטפל
```

## צעד 4: ציור התרשים

### עקרונות:
1. **זרימה משמאל לימין** (או מלמעלה למטה)
2. **שחקן אחד לכל שורה (Lane)**
3. **סמלים סטנדרטיים (BPMN)**

### תרשים בסיסי:

```
┌────────────────────────────────────────────────────────────────────┐
│                      תהליך אישור הוצאה                            │
├────────────┬───────────────────────────────────────────────────────┤
│            │                                                       │
│   עובד     │  ⭕ → [הגשת בקשה] ─────────────────────────→ ◎        │
│            │       + צירוף קבלה                         קבלת תשובה │
├────────────┼───────────────────────────────────────────────────────┤
│            │              ↓                                        │
│   מערכת    │       [בדיקת תקציב] → ◇ ──[יש תקציב]──→ [שליחה למנהל]│
│            │                       │                               │
│            │                       └──[אין]──→ [דחייה אוטומטית]   │
├────────────┼───────────────────────────────────────────────────────┤
│            │                                    ↓                  │
│   מנהל     │                              [בדיקה ואישור]           │
│            │                                    ↓                  │
│            │                                    ◇                  │
│            │                         [אישור]↙    ↘[דחייה]         │
├────────────┼───────────────────────────────────────────────────────┤
│            │                           ↓              ↓            │
│   כספים    │                     [ביצוע תשלום]   [הודעת דחייה]    │
│            │                           ↓              ↓            │
│            │                           ◉              ◉            │
└────────────┴───────────────────────────────────────────────────────┘
```

## צעד 5: אימות ותיעוד

### בדקו עם המבצעים:
- "זה מה שקורה באמת?"
- "מה חסר?"
- "איפה יש חריגים?"

### תעדו מידע נוסף:

```
┌─────────────────────────────────────────┐
│          תיעוד תהליך                    │
├─────────────────────────────────────────┤
│ שם: אישור הוצאה                         │
│ גרסה: 1.0                               │
│ תאריך: 15/01/2025                       │
│ בעלים: מחלקת כספים                      │
├─────────────────────────────────────────┤
│ מדדים נוכחיים:                          │
│ • זמן ממוצע: 4 ימים                     │
│ • אחוז דחיות: 15%                       │
│ • בקשות בחודש: 200                      │
├─────────────────────────────────────────┤
│ בעיות שזוהו:                            │
│ 1. עיכוב בשלב אישור מנהל               │
│ 2. קבלות לא ברורות מוחזרות             │
│ 3. אין מעקב אחרי בקשות תקועות          │
├─────────────────────────────────────────┤
│ הזדמנויות לשיפור:                       │
│ 1. תזכורות אוטומטיות למנהלים           │
│ 2. OCR לקריאת קבלות                    │
│ 3. דשבורד מעקב                          │
└─────────────────────────────────────────┘
```

## טיפים מהשטח

### עשו:
✅ התחילו עם "Happy Path" ואז הוסיפו חריגים
✅ שאלו "מה קורה כשזה נכשל?"
✅ צפו בעבודה אמיתית, לא רק הקשיבו
✅ שתפו את התרשים לאימות

### אל תעשו:
❌ אל תנסו לכלול הכל בתרשים אחד
❌ אל תתמקדו רק בטכנולוגיה
❌ אל תדלגו על שלב האימות
❌ אל תשכחו את החריגים
